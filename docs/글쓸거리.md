reflog show 했을 때 나오는 HEAD@{1} 차후에 사용가능한 문법이다.

name@{qulifier} 이것이 참조를 위해 reflog가 얘기하는 방법이다.

HEAD~2 이것은 두세저 부모 전을 얘기하는 것이다.
HEAD@{2} 는 커밋조차 아닐 수도 있다. 거슬러 올라가는 것이다. 동작의 기록?

git diff HEAD@{0} HEAD@{5} 

name@{qulifier}
git reflog master@{2.day.ago}
git reflog main@{3.minutes.ago}

git checkout master@{1.week.ago}

잃어버린 커밋 찾아오기.
git reset --hard main@{1}
